# Навигация

В приложении используется Jetpack Navigation Compose для управления переходами между экранами.

## Основные принципы

1.  **Централизация**: Все маршруты и логика видимости Bottom Bar сосредоточены в модуле `:shared:ui:navigation`.
2.  **Декларативность**: Видимость элементов интерфейса (Bottom Bar) определяется текущим маршрутом через константу `BOTTOM_BAR_ROUTES`.

## Компоненты навигации

### Bottom Bar (`AppBottomBar`)
- Глобальный элемент, размещенный в `Scaffold` внутри `MainActivity`.
- Отображается только на маршрутах, перечисленных в `BOTTOM_BAR_ROUTES` (находится в `BottomBarValues.kt`).
- Автоматически учитывает системные отступы (Navigation Bars) через `.navigationBarsPadding()`.

### Экран перехода (`TransitionScreen`)
- Используется для плавного переключения между тяжелым AR-режимом и обычным UI.
- Обеспечивает задержку (400 мс), необходимую для корректного завершения AR-сессии и освобождения ресурсов камеры.
- Визуально оформлен в стиле приложения (белый фон, фирменный индикатор загрузки).

### Обработка кнопки «Назад»
- В AR-режиме используется `BackHandler` для перехвата системных жестов и кнопок «Назад».
- Любой возврат из AR всегда проходит через `TransitionScreen` для предотвращения визуальных артефактов («рваной» анимации).

## Маршруты
- `catalog`: Главный экран со списком плитки.
- `ar`: Экран дополненной реальности.
- `transition`: Вспомогательный экран для переходов.
