# Контекст проекта: AR Plitka (Kotlin Multiplatform)

**Роль:** Ты — эксперт по Kotlin Multiplatform (KMP), Compose Multiplatform (CMP) и мобильной разработке (Android/iOS). Твоя задача — помочь мне реализовать экран деталей плитки.

**Цель проекта:** Приложение для измерения площади пола и визуализации раскладки плитки в AR.

**Текущее состояние (Готовность: ~70%):**
- **Архитектура:** Чистая архитектура (Clean Architecture), многомодульность.
- **DI:** Единый Koin для всех платформ (инициализация в `:shared:app`).
- **Ресурсы:** Compose Resources (`Res.string`, `painterResource`) полностью настроены и работают на Android и iOS.
- **Каталог:** Экран каталога (`:features:catalog`) готов, работает с моками и реальной сетью, поддерживает Pull to Refresh.
- **iOS AR:** Стабилен, реализована гибридная модель привязки к якорям, асинхронная сборка геометрии.
- **Документация:** Вся актуальная документация по архитектуре, модулям, DI и валидации находится в папке `docs/` на русском языке.

**История чатов для контекста:**
1.  **Основной чат (предыдущий):** `b6b7b439-bb40-426b-8ff3-519cf194107a` (здесь мы фиксили ресурсы iOS, настраивали Koin и обновляли документацию).

**Твоя задача: Реализация экрана деталей плитки (`Tile Details`)**

1.  **Модуль:** Создай (или используй существующий, если он есть) модуль `:features:tile-details` или расширь `:features:catalog`.
2.  **Domain:**
    - Реализуй UseCase для получения детальной информации о плитке по её ID.
    - Используй существующие модели из `:shared:tiles`.
3.  **Data:**
    - Добавь эндпоинт в `TilesApi` и мок-ответ в `:mock:tiles`.
4.  **Presentation:**
    - Создай `TileDetailsViewModel` с использованием `CustomResultModelDomain` и `CommonException`.
    - Реализуй UI на Compose Multiplatform:
        - Галерея изображений плитки.
        - Описание, характеристики (размеры, материал, производитель).
        - Выбор цвета/текстуры (если применимо).
        - Кнопка «Примерить в AR», которая передает выбранную плитку в AR-модуль.
5.  **Навигация:** Интегрируй переход из каталога в детали в модуле `:shared:ui:navigation`.

**Важные правила:**
- **Только русский язык** для документации и комментариев (см. `.cursor/rules/russian-docs.md`).
- Следуй правилам из `docs/tiles/CLEAN_ARCHITECTURE_GUIDE.md`.
- Используй UI-компоненты из `:shared:ui:kit`.
- Весь код должен быть KMP-совместимым и работать на обеих платформах.

Начни с анализа текущего состояния модуля `:shared:tiles` и предложи план реализации.
