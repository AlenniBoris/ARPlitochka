# Детальный обзор модулей (Этап 1: Tiles & Network)

В рамках первого этапа реализации системы работы с плиткой были созданы и настроены четыре ключевых модуля. Ниже приведено их детальное описание, структура и назначение.

## 1. Модуль `:shared:tiles`
**Назначение**: Основной бизнес-модуль для работы с данными плитки. Это KMP-модуль, код которого компилируется под Android и iOS.

### Внутренняя структура:
*   **`domain/model/`**: Чистые модели данных (`Tile`, `TileColor`, `TileVariant`, `TileUnit`). Не зависят от библиотек.
*   **`domain/repository/`**: Интерфейс `TilesRepository`. Определяет контракт получения данных.
*   **`domain/usecase/`**: UseCase-ы (например, `GetTilesUseCase`). Реализованы через паттерн **Прокси** (делегирование).
*   **`data/remote/`**: Описание API (`TilesApi`) и его реализация через Ktor (`TilesApiImpl`).
*   **`data/remote/dto/`**: Объекты для десериализации JSON (`TileDto`, `TileColorDto` и т.д.).
*   **`data/mapper/`**: Мапперы для преобразования DTO -> Domain и Throwable -> TileException.
*   **`data/repository/`**: `TilesRepositoryImpl` — реализация, которая управляет потоками данных, вызывает API и обрабатывает ошибки.
*   **`di/`**: (Android) Hilt-модуль для предоставления зависимостей.

---

## 2. Модуль `:network:core`
**Назначение**: Инфраструктурный модуль для всех сетевых взаимодействий в проекте.

### Ключевые компоненты:
*   **`HttpClientFactory`**: Централизованная точка создания `HttpClient` (Ktor). Здесь настраивается JSON-сериализатор, логирование и базовые таймауты.
*   **`CustomResultModelDomain<T, E>`**: Универсальный контейнер для результатов операций. Позволяет передавать либо данные, либо типизированную ошибку.
*   **`NetworkConfig`**: Общие константы (Base URL, настройки кэширования).

---

## 3. Модули `:mock:tiles` и `:mock:core`
**Назначение**: Обеспечение работы приложения без реального бэкенда.

### `:mock:tiles`:
*   **`assets/mock/tiles/catalog/all.json`**: Основной файл с данными плитки.
*   **`assets/mock/tiles/textures/`**: PNG-файлы текстур для AR.
*   **`TileAssets.kt`**: Объект-индекс для быстрого доступа к путям ассетов.
*   **`TileRoutes.kt`**: Определение маршрутов, которые должен перехватывать интерцептор.

### `:mock:core`:
*   Инфраструктура для чтения локальных файлов и подмены ответов Ktor (будет расширена на Этапе 2).

---

## Взаимосвязь модулей
1.  `TilesRepositoryImpl` (в `:shared:tiles`) использует `HttpClient` из `:network:core`.
2.  При включении моков, `HttpClient` начинает возвращать данные из `:mock:tiles`.
3.  Результат возвращается в UI в обертке `CustomResultModelDomain` из `:network:core`.
