# Детальный обзор модулей (Этап 1: Core, Tiles & Network)

В рамках реализации системы работы с плиткой и инфраструктуры приложения были созданы и настроены ключевые модули. Ниже приведено их детальное описание.

## 1. Модуль `:shared:core`
**Назначение**: Фундаментальное ядро приложения, доступное всем остальным модулям. Не зависит от внешних библиотек (кроме DataStore и Coroutines).

### Ключевые компоненты:
*   **`domain/model/`**: 
    *   `CustomResultModelDomain<T, E>`: Универсальная обертка для результатов операций (Success/Error).
    *   `CommonException`: Sealed interface для типизации доменных ошибок (Connection, Server и др.).
*   **`config/`**: 
    *   `AppConfig`: Модель настроек приложения (флаг моков, Base URL, задержка).
    *   `AppConfigManager`: Реактивный менеджер конфигурации на базе **Jetpack DataStore**.
*   **`domain/usecase/`**: Базовые UseCase-ы для управления настройками (`GetAppConfigUseCase` и др.).

---

## 2. Модуль `:shared:tiles`
**Назначение**: Бизнес-модуль для работы с данными плитки.

### Внутренняя структура:
*   **`domain/model/`**: Чистые модели данных (`Tile`, `TileColor` и т.д.).
*   **`domain/repository/`**: Контракт `TilesRepository`, возвращающий `CustomResultModelDomain`.
*   **`domain/mapper/`**: `ExceptionMapper.kt` для преобразования технических исключений в `CommonException`.
*   **`data/`**: Реализация репозитория, API (Ktor) и DTO.

---

## 3. Модуль `:network:core`
**Назначение**: Инфраструктурный модуль для сетевых взаимодействий.

### Ключевые компоненты:
*   **`HttpClientFactory`**: Создание и настройка `HttpClient` (Ktor) с поддержкой логирования и сериализации.
*   **`MockProvider`**: Интерфейс, через который сетевой слой взаимодействует с системой мокирования.

---

## 4. Модули `:mock:tiles` и `:mock:core`
**Назначение**: Обеспечение работы приложения без реального бэкенда.

### Ключевые компоненты:
*   **`AssetReader`**: Платформозависимое чтение JSON-файлов из `assets`.
*   **`MockRouteRegistry`**: Реестр соответствия URL-путей и файлов ассетов.
*   **`MockInterceptor`**: Плагин для Ktor, подменяющий ответы сервера данными из файлов.

---

## Взаимосвязь модулей
1.  Все модули зависят от **`:shared:core`** для использования общих моделей и конфигов.
2.  `TilesRepositoryImpl` (в `:shared:tiles`) использует `HttpClient` из `:network:core`.
3.  При включении моков в `AppConfig`, сетевой слой через `MockProvider` запрашивает данные у `:mock:core`.
