# ADR-001: Android Как AR-Референс

## Статус

Принято.

## Контекст

Проект поддерживает AR-поведение на Android и iOS. Сейчас Android содержит самый полный продуктовый AR-flow: поиск поверхности, постановка точек, замыкание контура, подтверждение, заливка, toggle плитки, смена типа плитки, поворот текстуры и reset.

iOS догоняет Android parity, используя ARKit/SceneKit и native bridge code.

## Решение

Android является продуктовым референсом для полного AR-flow, пока более новое продуктово-инженерное решение явно не изменит поведение для обеих платформ.

Когда Android и iOS отличаются:

1. Сначала проверяем Android-поведение.
2. Если Android-поведение совпадает с задокументированным продуктовым flow, iOS должен соответствовать ему.
3. Если Android-поведение неверное, обновляем канонические документы и исправляем обе платформы.

## Последствия

- iOS-реализация не должна придумывать отдельные AR UX rules.
- Shared KMP state и pure geometry предпочтительнее для parity-sensitive логики.
- Platform-specific rendering может отличаться внутри, но пользовательское поведение должно совпадать.
