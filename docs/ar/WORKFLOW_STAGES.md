# AR Workflow Stages (Стейт-машина процесса)

Для управления сложным пользовательским интерфейсом в AR (динамическое появление кнопок, смена инструкций, доступность действий) в проекте используется явная стейт-машина на базе этапов (`FloorWorkflowStage`).

## Зачем это нужно
Вместо проверки множества разрозненных флагов (`isPolygonClosed`, `isFinalized`, `points.isNotEmpty()`), логика UI и переходов между состояниями централизована. Это исключает некорректные состояния (например, показ кнопки «Добавить плитку» до того, как контур замкнут).

## Список этапов (`FloorWorkflowStage`)

1. **`INITIALIZING`**  
   Запуск AR-сессии. Камера еще не откалибрована, плоскости не найдены.  
   *UI:* Сообщение «Подождите...». Кнопка `Reset` (Пересканировать).

2. **`SEARCHING_FLOOR`**  
   AR-сессия активна, идет поиск горизонтальной поверхности.  
   *UI:* Инструкция «Наведите камеру на пол». Кнопка `Reset` (Пересканировать).

3. **`PLACEMENT_EMPTY`**  
   Поверхность пола найдена и стабилизирована. Ожидание первой точки.  
   *UI:* Кнопка `+` (Add Point), кнопка `Reset` (Пересканировать).

4. **`PLACEMENT_ACTIVE`**  
   Поставлено от 1 до N точек. Контур еще не замкнут.  
   *UI:* Кнопки `+`, `Undo`, `Reset`.

5. **`CONTOUR_CLOSED`**  
   Поставлено >= 3 точек, и текущий прицел (reticle) находится в зоне привязки (snap) к первой точке.  
   *UI:* Кнопка `Confirm` (Зеленая галочка), `Undo`, `Reset`.

6. **`CONTOUR_CONFIRMED`**  
   Пользователь нажал «Готово». Контур зафиксирован в мировых координатах.  
   *UI:* Большая кнопка `Add Tile` (Добавить плитку), кнопка `Reset`.

7. **`TILE_LAYOUT`**  
   Плитка отображена поверх пола.  
   *UI:* Панель настроек плитки (Поворот, Смена типа, Удаление плитки), кнопка `Reset`.

## Реализация

### Shared (Domain)
Логика переключения этапов находится в `FloorArController.kt` (метод `updateStage()`). Она одинакова для Android и iOS.

### Presentation
* **Android:** `ArActionButtons.kt` использует `when(uiState.stage)` для отрисовки соответствующего набора кнопок.
* **iOS:** `IosArScreen.ios.kt` аналогично использует `when(contourState.stage)`.

## Как добавлять новые этапы
1. Добавить новое значение в `FloorWorkflowStage` (Shared).
2. Обновить логику переходов в `updateStage()` в `FloorArController`.
3. Добавить отрисовку кнопок для нового этапа в `ArActionButtons` (Android) и `IosArScreen` (iOS).
