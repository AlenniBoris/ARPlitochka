# AR Plitka Android

Android-фронт приложения AR Plitka. Приложение использует камеру и ARCore, чтобы находить поверхность пола, позволять пользователю выделять контур зоны, визуализировать выбранную область и показывать текстуру плитки в реальном масштабе.

## Назначение Android-приложения

Android-часть отвечает за пользовательский AR-сценарий:

- проверка поддержки ARCore;
- запрос разрешения камеры;
- поиск горизонтальной поверхности пола;
- разметка пользовательского контура точками;
- отображение линий, длин сегментов, заливки и текстуры плитки;
- финализация выделенной зоны;
- подготовка данных, которые в будущем будут отправляться на backend.

Тяжелые расчеты раскладки плитки, работа с проектами, каталогом, базой данных и оптимизацией обрезков должны жить на сервере. План backend-части описан в `BACKEND_ROADMAP.md`.

## Gradle-модули

```text
AR Plitka
├── app
├── features
│   └── floor-detection
└── shared
    ├── ar
    └── ui
```

### `:app`

Application-модуль. Содержит точку входа приложения, Hilt application class, runtime-проверки разрешения камеры и поддержки ARCore.

Подробнее: `app/README.md`.

### `:features:floor-detection`

Основная функциональная фича. Содержит AR-экран, ViewModel, use case, repository, доменные модели и визуализацию выделенной зоны.

Подробнее: `features/floor-detection/README.md`.

### `:shared:ar`

Общие ARCore-расширения: фильтрация горизонтальных плоскостей, расчет площади плоскости и hit-test из центра экрана.

Подробнее: `shared/ar/README.md`.

### `:shared:ui`

Переиспользуемые Compose-компоненты: ретикл, статусная панель, blocking screen, debug panel и `UiText`.

Подробнее: `shared/ui/README.md`.

## Архитектурный Подход

Проект разделен на feature-модули и shared-модули:

- `app` не содержит бизнес-логику AR-разметки, а только собирает приложение и проверяет системные условия.
- `features/floor-detection` содержит самостоятельную фичу с разделением на `data`, `domain`, `presentation` и `di`.
- `shared/ar` содержит маленький набор переиспользуемых ARCore utilities.
- `shared/ui` содержит общие UI-компоненты без знания о конкретной фиче.

Текущее направление развития: сохранить Android как AR-фронт, а расчетную и серверную бизнес-логику вынести в отдельный backend-проект.

## Сборка и Запуск

Собрать debug-версию:

```powershell
./gradlew :app:assembleDebug
```

Установить на подключенное устройство:

```powershell
./gradlew :app:installDebug
```

Запустить установленное приложение:

```powershell
C:\Users\User\AppData\Local\Android\Sdk\platform-tools\adb.exe shell am start -n com.example.arplitka/com.example.arplitka.MainActivity
```

## Требования к Устройству

- Android 8.0+ (`minSdk = 26`)
- камера;
- поддержка ARCore;
- установленный/доступный Google Play Services for AR.

## Текущее Поведение

Пользователь открывает приложение, наводит камеру на пол, ставит точки по контуру зоны, видит соединяющие линии и подписи длин. Когда последняя точка подведена к первой, зона считается замкнутой и появляется возможность завершить процесс. После завершения отображается текстура плитки, повторяемая в реальном физическом масштабе.
